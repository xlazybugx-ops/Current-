# VEDRIVE — приложение для договоров аренды

PWA-приложение (работает как иконка на iPhone) для быстрого оформления договоров
аренды авто, ведения реестра, графика занятости и отчётности.

## Что умеет

- **Новый договор** — мастер из 5 шагов: даты → выбор свободного авто →
  цены и услуги (тянутся из прайса, поля редактируемые) → данные водителя →
  проверка → **«Подготовить договор»**.
- Автоматически присваивает **номер**, заносит в **реестр** и в **график аренды**.
- Выгрузка готового договора в **DOC** и **PDF** прямо на устройстве.
- **Реестр** договоров с поиском, просмотром, отменой/восстановлением.
- **График аренды** — диаграмма занятости по каждому авто, отметка «сегодня».
- **Отчёт** — KPI, доход по авто и по моделям, выгрузка реестра в CSV.
- **Автопарк** — добавление/редактирование машин и цен.
- Работает **офлайн**. База хранится на устройстве.
- Резервная копия базы: выгрузка/загрузка в файл (JSON).
- Необязательная **облачная синхронизация** между устройствами (см. ниже).

## 1. Публикация на GitHub Pages

1. Создайте репозиторий на GitHub, например `vedrive-app`.
2. Загрузите в него **все файлы из этой папки** (через сайт GitHub:
   `Add file → Upload files`, перетащите всё, `Commit`).
3. Откройте `Settings → Pages`.
4. В разделе **Build and deployment → Source** выберите `Deploy from a branch`,
   ветку `main`, папку `/ (root)`, нажмите `Save`.
5. Через минуту появится адрес вида
   `https://ваш-логин.github.io/vedrive-app/`.

## 2. Установка на iPhone (как приложение)

1. Откройте адрес из шага 1 в **Safari** на iPhone.
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой).
3. Выберите **«На экран „Домой"»** → `Добавить`.
4. На рабочем столе появится иконка **VEDRIVE** — открывается на весь экран,
   без адресной строки, работает офлайн.

## 3. Облачная синхронизация (необязательно)

GitHub Pages — статический хостинг без сервера, поэтому для синхронизации между
устройствами используется бесплатный **Google Apps Script**. Настройка ~10 минут.

### Шаг 1. Создайте таблицу и скрипт
1. Создайте новую Google-таблицу.
2. Меню `Расширения → Apps Script`.
3. Удалите содержимое и вставьте код из файла **`apps-script.gs`** (в этой папке).
4. Нажмите `Сохранить`.

### Шаг 2. Опубликуйте как веб-приложение
1. Кнопка `Развернуть → Новое развёртывание`.
2. Тип: **Веб-приложение**.
3. «Выполнять от имени»: **Я**.
4. «У кого есть доступ»: **Все** (Anyone).
5. `Развернуть`, разрешите доступ, скопируйте **URL веб-приложения**
   (заканчивается на `/exec`).

### Шаг 3. Подключите в приложении
1. В приложении откройте вкладку **Авто** → раздел «Синхронизация».
2. Вставьте URL, нажмите **«Сохранить адрес»**.
3. **«⬆ В облако»** — отправить текущую базу. На другом устройстве —
   **«⬇ Из облака»**. После настройки новые договоры выгружаются автоматически.

> Безопасность: ссылка даёт доступ к чтению/записи базы. Не публикуйте её.
> Для приватного доступа можно ограничить «У кого есть доступ» и хранить ссылку
> только на своих устройствах.

## Файлы проекта

| Файл | Назначение |
|------|-----------|
| `index.html` | Точка входа |
| `style.css` | Оформление |
| `data.js` | Прайс, автопарк, стартовые договоры (из Excel) |
| `db.js` | База данных (localStorage), доступность авто, синхронизация |
| `doc.js` | Генерация договора в DOC и PDF |
| `app.js` | Логика экранов и мастера |
| `manifest.webmanifest`, `sw.js` | PWA: иконка, офлайн |
| `icon-*.png` | Иконки приложения |
| `apps-script.gs` | Код для облачной синхронизации (Google) |
